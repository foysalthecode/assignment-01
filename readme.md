#ব্লগ পোষ্ট

১। টাইপস্ক্রিপ্টে ইন্টারফেস এবং টাইপস প্রকারের মধ্যে কিছু পার্থক্য কী?

-- ইন্টারফেস মুলত অবজেক্ট এর স্ট্রাকচার ডিফাইন করার জন্য ব্যবহার করা হয় ।এগুলা বিশেষ কারন এগুলা ক্লাস এর সাথে সহজে ইন্টিগ্রেড করতে পারে ।কোনো ক্লাস চাইলতে ইন্টারফেস ইমপ্লিমেন্ট করে তার নির্দিষ্ট আকার অনুসরন করতে পারে।আবার একই নামে ২ বার ইন্টারফেস লিখতে টাইপস্ক্রিপ্ট সেগুলা আটো মারজ করে দেয়।
তাই বড় প্রজেক্ট এ বিভিন্ন জায়গায় ইন্টারফেস এক্সপেন্ড করা সহজ হয়।সাধারণ অবজেক্ট স্ট্রাকচার, মডেল, এপি আই রেস্পন্স, এবং reusable shape define করার ক্ষেত্রে ইন্টারফেস সবচেয়ে ক্লিন

--টাইপ্স আরো ফ্লেক্সিবল এবং ্পাওয়ারফুল একটা সিস্টেম। এগুলো শুধু ুঅবজেক্ট নয় , প্রিমিটিব (স্ট্রিং, নাম্বার,ইউনিয়ন (A | B),ইন্টারসেকসন (A & B), tuple, function type) সব কিছু ্ডিফাইন করতে পারে। তবে type-এর একটা লিমিটেশন হলো: এগুলো merge হয় না, অর্থাৎ এক নামের দুইটা টাইপ লিখলে এরোর দেবে। কিন্তু extension করা যায় & operator দিয়ে। এই কারণে টাইপ খুব ইউজফুল যখন complex type logic, union, অথবা multiple possible shapes define করার দরকার হয়।

২। টাইপস্ক্রিপ্টে keyof কীওয়ার্ডের ব্যবহার কী? একটি উদাহরণ দিও।

--keyof মুলত কোনো অবজেক্ট টাইপ এর সবগুলো প্রোপার্টি নামকে (keys) একটা union type হিসাবে ধরে। টাইপিং এরোর কমে,সেফটি বাড়ে।

উদাহরণ --

interface User {
name: string;
age: number;
country: string;
}

type Userkeys = keyof User;

let Key: Userkeys;

খানে User নামে একটা interface আছে, যার তিনটা property
এখানে key হলো name,age,country । মানে এই তিনটার বাইরে অন্য কিছু assign করলে error দিবে।
